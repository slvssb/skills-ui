## Design Context

### Users
Developers use this app to manage installed skills in a local, utility-first workflow. The core job is to help them inspect, find, add, delete, update, and eventually group installed skills by git repository without friction.

### Brand Personality
Helpful, minimalistic, powerful. The interface should feel organized, fast, and dependable for technical users who want clarity over decoration.

### Aesthetic Direction
Desktop-native utility UI with deliberate structure, quiet visual hierarchy, and strong information density. The app should avoid clumsy layouts, weak organization, and generic filler styling.

### Design Principles
1. Put management actions and skill state ahead of passive reading.
2. Keep the layout minimal, but make structure and grouping immediately legible.
3. Prefer native-feeling restraint over decorative styling.
4. Make power-user workflows discoverable without making the app feel busy.
5. Use hierarchy to clarify what can be acted on now versus what is merely informative.
